    Основи

    C# — це мова зі статичною типізацією, що означає: кожна змінна має тип, відомий під час компіляції. Присвоєння значення імені називається оголошенням змінної. Її можна оголосити, явно вказавши тип, або дозволити компілятору вивести тип автоматично на основі присвоєного значення (це називається type inference — виведення типу). Тому обидва оголошення еквівалентні:

    int explicitVar = 10; // Явно вказаний тип
var implicitVar = 10; // Тип виведений компілятором

    Оновлення значення змінної здійснюється оператором =. Після визначення тип змінної ніколи не може змінитися.

    var count = 1; // Початкове значення
count = 2;     // Нове значення

// Помилка компіляції:
// count = false;

    C# — об’єктно-орієнтована мова, тому всі функції мають бути оголошені всередині класу. Класи оголошуються за допомогою ключового слова class. Об’єкти (екземпляри) створюються за допомогою оператора new:

    class Calculator
{
    // ...
}

var calculator = new Calculator();

    Функція всередині класу називається методом. Метод може мати нуль або більше параметрів. Кожен параметр повинен мати явно вказаний тип — виведення типу не застосовується. Так само потрібно явно вказати тип значення, що повертається методом. Повернення значення виконується через ключове слово return. Щоб метод був доступний з інших файлів, потрібно додати модифікатор public.

    class Calculator
{
    public int Add(int x, int y)
    {
        return x + y;
    }
}

    Методи викликаються через синтаксис крапки, передаючи аргументи:

    var calculator = new Calculator();
var sum_v1 = calculator.Add(1, 2);
var sum_v2 = calculator.Add(x: 1, y: 2);

    Область видимості в C# визначається фігурними дужками { }.

    C# підтримує два типи коментарів: однорядкові починаються з //, а багаторядкові записуються між /* і */.

    Інструкції

    Дівчина Лусіана вже їде додому, а він так і не приготував вечерю до річниці!

    У цьому завданні треба допомогти Лусіану приготувати вишукану лазанью за його улюбленою кулінарною книгою.

    Усього є чотири завдання, кожне з яких стосується часу приготування.

    1️⃣ Визнач очікуваний час у духовці

    Створіть метод Lasagna.ExpectedMinutesInOven(), який не приймає параметрів і повертає кількість хвилин, скільки лазанья повинна бути в духовці. Згідно з рецептом — 40 хвилин.

    var lasagna = new Lasagna();
lasagna.ExpectedMinutesInOven();
// => 40

    2️⃣ Обчисли залишковий час у духовці

    Створіть метод Lasagna.RemainingMinutesInOven(), який приймає фактичний час у духовці і повертає, скільки хвилин залишилося, базуючись на очікуваних 40 хвилинах.

    var lasagna = new Lasagna();
lasagna.RemainingMinutesInOven(30);
// => 10

    3️⃣ Обчисли час підготовки

    Створіть метод Lasagna.PreparationTimeInMinutes(), який приймає кількість шарів у лазаньї. Кожен шар займає 2 хвилини підготовки.

    var lasagna = new Lasagna();
lasagna.PreparationTimeInMinutes(2);
// => 4

    4️⃣ Обчисли загальний витрачений час

    Метод Lasagna.ElapsedTimeInMinutes() приймає два параметри:

    1. кількість шарів
    2. час у духовці

    Потрібно повернути сумарний час: підготовка + час у духовці.

    var lasagna = new Lasagna();
lasagna.ElapsedTimeInMinutes(3, 20);
// => 26
